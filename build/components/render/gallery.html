Moving 2D content to a 3D world

The VR/AR web seems to be the latest buzz word in the web community.

At I/O 2016 Google made several big announcements about the technology... Older products and frameworks are still going strong but the competition keeps getting stronger all the time

The most interesting metaphor is how we read online... this goes beyond Nielsen's people only scan online content but tries to answer a more fundamental question of building initerfaces and new metaphors for reading content online.

Multi platform support is the sticking point for development.

None of the big desktop vendors support Mac as a platform for either development or playback. Trying to figure out how much of a sore subject this will become moving forward. None of the major HMD support the Mac as a platform so we can't test those devices for that platform... how do we ensure that our solution displays properly on those devices?

See Josh Carpenter's SFHTML5 Presentation on VR UX/UI from 2015 to provide a contrast to where we are now.

Possible deliverables may include:

  • Selection of tools and technologies to accomplish the task
  • Ideas for how to dusplay 2D content in a 3D environment
  • Code to accomplish this display

Progressive and subcompact web reading experiences

The web has become as good as, if not better than native experience. I'm looking at what it would take to build a web based reading experience incorporating newer technologies in the CSS, Web APIs and HTML5 arenas.

At the same time I seek to build a toolbox for these applications: What tools to use and how to best leverage what the web can do best over native applications and how to make the web more app like without loosing what makes it great :-). I also want to document the philosophy behind these reading experiences (I deliberately choose not to call them books.... They are more and they are less than printed books and they coexist in a continuum)

Two pieces influenced the thinking on this project: Alex Russell's Progressive Web Apps: Escaping Tabs Without Losing Our Soul and Craig Mod's Subcompact Publishing

GPL and why I won't release my Wordpress code

I don't like GPL but most of the CMS/Blogging software I use is licensed under it so I'm stuck if I want to do any development work I have to release my code under GPL. Automattic/Wordpress.com will not support or advertise any theme or plugin that is not released fully under GPL. That raises questions whether the 'premium theme' creators distributors are in compliance with the license and how do we interpret the license itself.

There is also the crappy attitude and tactic Automattic followed against Chris Pearson... they not only went after the domain that he did have a right to but went after his copyrights and his livelyhood.

Beyond philosophical questions there is the issue of GPL's viral nature. If you use GPL code you're required to release as open source... ideally under GPL but any other compatible license will do in a pinch, unless your Matt Mullenweg and Automattic... Their attitude puts developers like me in a bind... if you want to release your code through them use their license, even if you'd normally release the code under a more permissive license than GPL is. Thanks but no thanks.

Moving 2D content to a 3D world

The VR/AR web seems to be the latest buzz word in the web community.

At I/O 2016 Google made several big announcements about the technology... Older products and frameworks are still going strong but the competition keeps getting stronger all the time

The most interesting metaphor is how we read online... this goes beyond Nielsen's people only scan online content but tries to answer a more fundamental question of building initerfaces and new metaphors for reading content online.

Multi platform support is the sticking point for development.

None of the big desktop vendors support Mac as a platform for either development or playback. Trying to figure out how much of a sore subject this will become moving forward. None of the major HMD support the Mac as a platform so we can't test those devices for that platform... how do we ensure that our solution displays properly on those devices?

See Josh Carpenter's SFHTML5 Presentation on VR UX/UI from 2015 to provide a contrast to where we are now.

Possible deliverables may include:

  • Selection of tools and technologies to accomplish the task
  • Ideas for how to dusplay 2D content in a 3D environment
  • Code to accomplish this display

Progressive and subcompact web reading experiences

The web has become as good as, if not better than native experience. I'm looking at what it would take to build a web based reading experience incorporating newer technologies in the CSS, Web APIs and HTML5 arenas.

At the same time I seek to build a toolbox for these applications: What tools to use and how to best leverage what the web can do best over native applications and how to make the web more app like without loosing what makes it great :-). I also want to document the philosophy behind these reading experiences (I deliberately choose not to call them books.... They are more and they are less than printed books and they coexist in a continuum)

Two pieces influenced the thinking on this project: Alex Russell's Progressive Web Apps: Escaping Tabs Without Losing Our Soul and Craig Mod's Subcompact Publishing

GPL and why I won't release my Wordpress code

I don't like GPL but most of the CMS/Blogging software I use is licensed under it so I'm stuck if I want to do any development work I have to release my code under GPL. Automattic/Wordpress.com will not support or advertise any theme or plugin that is not released fully under GPL. That raises questions whether the 'premium theme' creators distributors are in compliance with the license and how do we interpret the license itself.

There is also the crappy attitude and tactic Automattic followed against Chris Pearson... they not only went after the domain that he did have a right to but went after his copyrights and his livelyhood.

Beyond philosophical questions there is the issue of GPL's viral nature. If you use GPL code you're required to release as open source... ideally under GPL but any other compatible license will do in a pinch, unless your Matt Mullenweg and Automattic... Their attitude puts developers like me in a bind... if you want to release your code through them use their license, even if you'd normally release the code under a more permissive license than GPL is. Thanks but no thanks.
